# Code of Conduct

## 1. Purpose
This repository aims to provide a helpful, clean, and collaborative space for learning Data Structures & Algorithms (DSA).  
To maintain a positive environment, all contributors and participants are expected to follow this Code of Conduct.

## 2. Expected Behavior
Participants are expected to:
- Show respect, professionalism, and kindness in all interactions.
- Use inclusive and welcoming language.
- Provide constructive feedback when discussing code or solutions.
- Credit external resources when applicable.
- Keep discussions focused on learning and improvement.

## 3. Unacceptable Behavior
The following behavior is not allowed:
- Harassment, discrimination, or personal attacks.
- Using someone elseâ€™s code without attribution.
- Sharing harmful, unsafe, or malicious content.
- Spamming or derailing discussions.
- Any behavior that makes others feel uncomfortable or unwelcome.

## 4. Contribution Guidelines
Contributors should:
- Submit clean and readable code.
- Follow the existing folder structure and naming conventions.
- Add comments when necessary for clarity.
- Avoid committing unrelated or unnecessary files.
- Use issues and pull requests responsibly and professionally.

## 5. Reporting Violations
If you notice behavior that violates this Code of Conduct:
- Open an issue describing the problem, **OR**
- Contact the repository owner/maintainer directly.

All reports will be taken seriously and investigated.

## 6. Enforcement
The repository owner may:
- Issue warnings for inappropriate behavior.
- Edit or remove contributions that violate this Code.
- Restrict or ban contributors if necessary.

## 7. Attribution
This Code of Conduct is adapted from the Contributor Covenant (v2.1) for use in this personal DSA repository.
